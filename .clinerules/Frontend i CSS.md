# Tabik — frontend i CSS

- Traktuj CSS jako część architektury, nie jako miejsce losowych dopisków.
- Przed zmianą UI znajdź wszystkie klasy, selektory i komponenty powiązane z danym widokiem.
- Usuwając element z interfejsu, usuń też martwe style i nieużywane fragmenty JS.
- Nie naprawiaj CSS przez nadmierne `!important`, chyba że to izolowana, uzasadniona awaria.
- Preferuj spójne nazewnictwo klas i grupowanie stylów według obszaru interfejsu.
- Po zmianach UI sprawdź: desktop, wąski ekran, hover/focus, aktywną zakładkę i brak martwych linków.
